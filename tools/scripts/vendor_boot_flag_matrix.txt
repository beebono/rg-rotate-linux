HS-init boot image: tools/spd_dump/boot_custom.img

These are vendor_boot images, because the panel flags live in the DTB carried by
vendor_boot_a. The boot image is shared across the matrix and remains:
tools/spd_dump/boot_custom.img

Observed behavior so far:
- `5`, `1`, and `1025` are non-burst modes and fail during DSI host setup with
  `current resolution can not be set`. They are useful diagnostics, but are not
  valid candidates at the current 2-lane / 551.488 Mbps / 38.4 MHz timing budget.
- Burst-family variants continue through normal controller programming and remain
  the best candidates for a visible panel result.

Flash template:
./spd_dump --wait 300 keep_charge 1 \
  fdl ../../device/stock/fw/extracted/fdl1-dl.bin 0x5500 \
  fdl ../../device/stock/fw/extracted/fdl2-dl.bin 0x9EFFFE00 \
  write_part vendor_boot_a vendor_boot_custom_<FLAG>.img

Suggested test order:
1. 3

4. 1539
5. 5

7. 1025
